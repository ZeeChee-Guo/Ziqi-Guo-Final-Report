\chapter{Related Work}

This chapter reviews prior work related to time-series anomaly detection and concept drift adaptation. Existing subsequence anomaly detection methods provide strong foundations for detecting unusual time-series regions. After applying the algorithms introduced in this chapter, each point in the time series data will be assigned an anomaly score. The higher the score, the more likely it is to be an anomaly. This section briefly explains the key ideas, advantages and disadvantages of the NormA, SAND, DAMP and Andri algorithms.

\section{NormA}

NormA is an unsupervised method for subsequence anomaly detection in time series \citep{boniol2021norma}. The key idea is that normal behavior tends to appear repeatedly over time. NormA first extracts subsequences from the time series and uses offline clustering to identify representative groups of recurring subsequences. The representative subsequences from these clusters form a normal model that characterizes the dominant normal behavior in the data.

During the detection process, the candidate subsequences are compared with the normal patterns that have been learned in the model. When a subsequence differs significantly from the representative normal pattern, it will receive a higher anomaly score. In this way, NormA regards the anomaly as a subsequence that deviates from the repetitive patterns discovered from the data.

NormA learns the regular patterns that can summarize repetitive shapes. Therefore, the abnormal score is calculated based on normal behavior. This makes NormA very useful when the data contains repetitive normal patterns, as this method can capture the main structure of the time series and use it as a stable reference standard for subsequent anomaly detection.

However, NormA builds its normal model offline and treats the learned patterns as a relatively fixed description of normal behavior. This assumption may not hold in the presence of concept drift, where normal patterns evolve over time. Patterns that are normal only during a specific period may be underrepresented in the learned model, while outdated patterns can keep affecting detection results. Consequently, changes caused by concept drift may be incorrectly identified as anomalies. To address this limitation, AnDri introduces a dynamic normal model that adapts to changing patterns and updates the set of active normal behaviors over time \citep{park2025adaptive}.


\section{SAND}

SAND is an unsupervised online method for streaming subsequence anomaly detection \citep{boniol2021sand}. SAND is designed for data streams, where new values arrive continuously and the model must be updated over time. SAND maintains a model of normal behavior and score new subsequences according to their distance from this model.

SAND first builds an initial normal model from an initial batch of the time series. It extracts subsequences from the batch and applies a time-series clustering method based on \(k\)-Shape to group similar subsequences. The centroids of these clusters represent common subsequence shapes and are stored as the current model of normal behavior. When a new batch arrives, SAND clusters the new subsequences and compares the new cluster centroids with the existing model. If a new cluster matches an existing normal pattern, the model can be updated. If it is far from the existing model, the corresponding subsequences receive higher anomaly scores.

SAND supports streaming anomaly detection. Since the model is updated batch by batch, SAND can adapt to changes in the data distribution. This makes it suitable for real-time monitoring tasks. However, SAND mainly adapts the normal model through streaming cluster updates. It does not explicitly represent normal patterns as active, inactive, or newly added concepts.

\section{DAMP}

DAMP is an online anomaly detection method based on the Matrix Profile and discord discovery \citep{lu2022damp}. DAMP detects abnormal situations by identifying subsequences that are significantly different from those previously observed. For each incoming subsequence, DAMP searches for its closest and non-overlapping match in the past. If the most recent previous subsequence is still far apart, the current subsequence is considered abnormal. This distance is used as the abnormal score. In the terminology of matrix contours, DAMP focuses on the left inconsistent items, that is, a subsequence is considered abnormal because it has no approximate matching items in the earlier subsequences of the flow. A larger left nearest neighbor distance indicates that the subsequence is likely abnormal.

DAMP is designed for large-scale and rapidly arriving time series streams, and its performance in anomaly detection far exceeds that of traditional difference search methods. This makes DAMP a useful benchmark for evaluating whether a system can efficiently detect anomalous subsequences in an online environment.

However, DAMP does not explicitly model normal behavior as a set of evolving patterns. When a new normal pattern appears because of concept drift, it may initially look like a discord because it has no close match in the past. As a result, DAMP may assign high anomaly scores to drift-driven changes before the new behavior becomes repeated enough to look normal.

\section{AnDri}

AnDri is an anomaly detection algorithm designed for time series with concept drift \citep{park2025andri,park2025adaptive}. Unlike the methods that use fixed conventional models, Andri assumes that normal behavior changes over time. Therefore, it maintains a dynamic conventional model, in which the regular patterns can be in an active state, an inactive state, or newly added as the time sequence changes.

This algorithm first learns normal patterns from the initial training region. It uses the adjacent hierarchical clustering (AHC) method to group similar subsets while simultaneously considering their temporal proximity. This allows the algorithm to identify normal patterns occurring within a given period, rather than assuming that all normal patterns must appear frequently throughout the entire time series. During the detection process, each new subseries is compared to the currently active normal patterns. If the subsequence is similar to the active pattern, it is regarded as part of the current normal behavior. If an old pattern becomes relevant again, it can be reactivated. If recent subsequences do not match any of the existing active patterns but persist within the recent window, Andri may consider them as new normal patterns caused by concept drift. Otherwise, these subsequences are regarded as abnormal candidate patterns. The anomaly score is calculated based on the distance between the current subsequence and the most similar active normal pattern. This is different from methods such as NormA, which compare subsequences with a more static set of learned normal patterns. By focusing on the active normal behaviors, Andri can more accurately distinguish between true abnormal behaviors and changes in normal behaviors caused by drift.

The main advantage of Andri is to explicitly model the lifecycle of normal patterns. This makes it more suitable for time series data where normal behaviors change, disappear, or reappear over time. In this report, Andri is regarded as the algorithm foundation of the Andri system, while subsequent chapters focus on the implemented interactive workflow, visualization, threshold adjustment, and system deployment.

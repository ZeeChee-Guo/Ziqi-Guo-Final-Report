\chapter{Methodology}

This chapter explains the methodology used to refine anomaly thresholds in this project. It first discusses why determining the final threshold for anomaly scores is challenging after scores have been assigned to each data point. A threshold that is too strict may incorrectly classify normal behavior as anomalous, while a threshold that is too loose may fail to detect true anomalies. Based on this motivation, Section 4.2 proceeds to describe the proposed threshold optimization process. It explains how initial user labels and the maximum anomaly proportion are used, how the system fits the distribution of confirmed anomaly scores, how uncertain points are selected for additional user review, and how the threshold is iteratively updated before producing the final anomaly labels.

\section{Problem Definition}

Let \(T=[x_1,x_2,\ldots,x_n]\) be a uni-variate time series, where \(x_i\) is the observed value at time index \(i\). The goal of anomaly detection is to decide whether each time point is normal or anomaly. Anomaly detection method first assigns each time point an anomaly score \(s_i\), and a larger score means that the point is more suspicious.

The anomaly detection algorithms transforms the original time series into a anomaly-score sequence:
\[S=[s_1,s_2,\ldots,s_n].\]
Here, each point \(i\) has an anomaly score \(s_i\).

After the anomaly scores are generated, a threshold is applied to decide the final anomaly label (anomaly or normal points)
\[
\hat{y}_i =
\begin{cases}
1, & \text{if } s_i > \theta,\\
0, & \text{otherwise,}
\end{cases}
\]
where \(\hat{y}_i=1\) means that point \(i\) is predicted as anomaly, and \(\hat{y}_i=0\) means that it is predicted as normal point. This workflow is common in time-series anomaly detection. Time series anomaly detect methods produce anomaly scores. Final anomaly labels depends on how the threshold is selected.

Concept drift complicates threshold selection because normal behavior changes over time. A fixed threshold may therefore produce false positives or miss true anomalies. AnDri system addresses this problem by supporting interactive threshold adjustment. The user can review candidate anomalous points, provide feedback. So that the system can refine the threshold according to the user feedback to convert anomaly scores into final labels. This feedback process is important in the system because both AnDri and the baseline methods generate point anomaly scores, and a threshold must be proposed and refined to predict the final decision.

Figure~\ref{fig41} illustrates this problem formulation. The upper plot shows the original time series. The lower plot shows the anomaly score of each time point. The threshold is applied to the anomaly-score curve. At the same time, this picture also illustrates that a high threshold may miss true anomalies because only the anomalies with most extreme scores are detected. A low threshold may generate many false positives by labeling normal points as anomalies. The target threshold achieves a balance between detecting true anomalies and limiting false alarms.

\section{Active Learning-Based Threshold Refinement}
It can be seen in Section 4.1 that choosing an appropriate threshold is a challenging task. Therefore, this project attempts to combine the user's expertise to design a human-in-the-loop process, aiming to obtain a better final threshold. This section provides a detailed description of the specific details of the active learning algorithm.


\subsection{Method Overview}

The anomaly detection algorithm generates anomaly scores for each data point. After that, a threshold needs to be selected to determine which points are anomalies. As explained in the previous section, the selection of the threshold is a challenging task. Therefore, this project uses an active-learning-based threshold refinement workflow that combines detector scores, partial user labels, Bayesian mixture fitting, and iterative candidate review. 

The workflow begins with the user's selected training region, an initial set of marked anomaly points, and a maximum anomaly proportion \(\rho\). For baseline detectors such as NormA, SAND, and DAMP, the system estimates one global threshold from the training region. For AnDri, the same threshold estimation process is applied within each normal-pattern group, so that different normal behaviors have different decision thresholds. 

The active-learning loop is triggered when the currently marked anomaly scores do not support a stable threshold. In that case, the system selects informative candidates: possible low-score mislabels and unlabeled points near the uncertain boundary between score groups. The user's feedback updates the label flags, and the Bayesian mixture model is fitted again using the revised confirmed anomaly scores. The loop continues until the score distribution is stable enough to set the final threshold. Figure~\ref{fig:42} summarizes this process. The next subsection presents the detailed illustration of this process.

\subsection{Dynamic Threshold Setting with User Feedback}

This part provides a detailed explanation of exactly how to utilize user feedback to select a better threshold.

Let \(\rho \in [0,1]\) be the maximum allowable anomaly proportion specified by the user. This parameter provides a proportion-based constraint during threshold estimation. For example, when \(\rho=0.05\), it means that at most \(5\%\) of points with the highest anomaly scores would be classified as anomalies in the final stage.

At the beginning of the interaction, the user marks anomaly points. The system maintains a label state
\[
f_i =
\begin{cases}
0, & \text{unlabeled},\\
1, & \text{user-confirmed anomaly},\\
2, & \text{user-confirmed normal}.
\end{cases}
\]

Only the scores of user-confirmed anomalies are used to estimate the anomaly-score distribution. Points with \(f_i=2\) are not used as normal training examples for fitting the Bayesian model; they are removed from the candidate anomaly set and excluded from future uncertain-candidate queries. They are regarded as erroneous labels made by the users.

Given the current confirmed anomaly set \(L=\{i\in A\mid f_i=1\}\), let \(S_L=\{s_i\mid i\in L\}\) denote the anomaly scores of these user-confirmed anomaly points. The backend fits a two-component Bayesian Gaussian mixture model to \(S_L\):
\[p(s)=\sum_{k=1}^{2}\pi_k\mathcal{N}(s;\mu_k,\sigma_k^2).\]

Here, \(s\) is an anomaly score value, and \(p(s)\) denotes the probability density of the scores assigned to the user-confirmed anomalies. After fitting, the model estimates the mixture weights \(\pi_1,\pi_2\), the component means \(\mu_1,\mu_2\), and the component variances \(\sigma_1^2,\sigma_2^2\). The weight \(\pi_k\) indicates the proportion of the confirmed anomaly-score distribution explained by component \(k\). For example, after fitting, suppose the model may estimate two active components such as
\[ \pi_1=0.45,\quad \mu_1=0.70,\quad \sigma_1^2=0.01, \]
and
\[ \pi_2=0.55,\quad \mu_2=0.90,\quad \sigma_2^2=0.005. \]
This result means that approximately \(45\%\) of the confirmed anomaly-score distribution is centered around score \(0.70\), while approximately \(55\%\) is centered around score \(0.90\). Since both components have meaningful mixture weights, the system treats the confirmed anomaly scores as two competing score groups rather than one stable distribution. In this case, the threshold is considered uncertain, and additional candidate points are selected for user review.


A component is treated as active only when its mixture weight is larger than a small weight threshold. If the Bayesian mixture has only one active component, the marked anomaly scores are considered stable. In this case, the confirmed anomaly scores are treated as being generated from one approximately Gaussian score distribution. Therefore, \(\mu_a-\sigma_a\) is used as a lower boundary for this stable anomaly-score component. Under a normal distribution, most values lie close to the mean; in particular, about \(84\%\) of the values are larger than \(\mu_a-\sigma_a\). Since points are predicted as anomalies when their scores are above the threshold, this lower boundary keeps most scores from the stable anomaly component while excluding the lower tail, which may contain uncertain or mislabeled points. The threshold is then set as
\[\theta=\max(\theta_{\rho},\mu_a-\sigma_a),\]
where \(\mu_a\) and \(\sigma_a\) are the mean and standard deviation of the stable active anomaly-score component. The maximum operation ensures that the final threshold is not lower than the threshold implied by the maximum anomaly proportion \(\rho\).

If two active components remain, the system regards the current threshold as uncertain. It then selects candidate points for user review in two ways. First, if a user-marked anomaly has a score below \(\theta_{\rho}\), the system asks the user whether this point should remain an anomaly. This step removes possible low-score mislabels. Second, the system finds the intersection region between the two active Gaussian components and asks the user to confirm unlabeled points near this uncertain boundary. If the user confirms a candidate, its flag becomes \(f_i=1\); if the user rejects it, its flag becomes \(f_i=2\). After each review round, the Bayesian mixture is refitted and the threshold is updated.

For the baseline detectors, including NormA, SAND, and DAMP, this process produces one global threshold. For AnDri, the same threshold-setting procedure is applied separately to each normal-pattern group in the selected training region. Let \(g(i)\) be the normal-pattern index assigned to point \(i\). For pattern \(k\), the training subset is
\[ A_k=\{i\in A\mid g(i)=k\}. \]
The system estimates a pattern-specific threshold \(\theta_k\) using only the scores and user feedback associated with \(A_k\). The final AnDri prediction is
\[
\hat{y}_i =
\begin{cases}
1, & s_i>\theta_{g(i)},\\
0, & s_i\leq \theta_{g(i)}.
\end{cases}
\]
If a test point belongs to a normal pattern that was not present in the selected training region, the system assigns the threshold of the most similar trained normal pattern, measured by Euclidean distance between normal-pattern subsequences.

\chapter{Related Work}

This chapter reviews prior work related to time-series anomaly detection and concept drift adaptation. Existing subsequence anomaly detection methods provide strong foundations for detecting unusual time-series regions. In particular, this chapter focuses on subsequence anomaly detection methods used as baselines in the AnDri work, including NormA, SAND, and DAMP, and then discusses concept drift detection methods and benchmark resources. 

\section{Time-Series Anomaly Detection}

Time-series anomaly detection aims to identify observations or subsequences that deviate from expected temporal behavior. General anomaly detection surveys classify anomalies according to their relationship with the data distribution and surrounding context \citep{chandola2009anomaly}, while time-series-specific evaluations show that performance can vary greatly across datasets, anomaly types, and evaluation protocols \citep{schmidl2022anomaly}. In subsequence anomaly detection, the object of interest is not only a single point but a local segment whose shape, frequency, amplitude, or duration is unusual.

A major family of time-series anomaly detection methods is based on subsequence similarity. The Matrix Profile provides an efficient way to compute nearest-neighbor distances among subsequences and has been widely used for motif and discord discovery \citep{yeh2016matrix}. A motif is a repeated subsequence, while a discord is a subsequence that is far from its nearest neighbor. This formulation is powerful because it is domain-agnostic and does not require a large labeled training set. However, discord-based reasoning often assumes that normal behavior is relatively stable or sufficiently repeated across the available data. When the time series contains concept drift, a new normal pattern may appear rare at first and may therefore be confused with an anomaly.

\section{Normal-Model-Based Detection: NormA}

NormA is an unsupervised and scalable subsequence anomaly detection method for large data series \citep{boniol2021norma}. Instead of searching only for isolated discords, NormA constructs a normal model from representative subsequences and detects anomalies according to their dissimilarity from this model. The method is attractive because it is domain-agnostic, supports recurrent anomalies, and does not require manual labels. It also formalizes the idea that normality can be represented by recurrent subsequences rather than by individual point values.

The limitation of NormA for the AnDri setting is that its normal model is primarily learned from the available historical series and is not explicitly managed as a changing set of active and inactive patterns. When normal behavior changes over time, a subsequence may be abnormal relative to the recent context but similar to an older normal pattern, or it may be a newly emerging normal pattern that has not yet become frequent globally. In these cases, a global normal model can blur the difference between anomaly and drift. AnDri differs by treating normal patterns as dynamic: patterns may become active, inactive, or newly added as the series evolves \citep{park2025adaptive}. This design allows anomaly scores to be computed relative to currently relevant normal patterns rather than all historical patterns.

\section{Streaming Detection: SAND and DAMP}

SAND extends subsequence anomaly detection to streaming data \citep{boniol2021sand}. It incrementally maintains a model of normal behavior and updates the model as new data arrives. This is important because many real applications, such as sensor monitoring and system logs, require detection before the full time series is available. SAND is stronger than purely offline methods in settings where the data distribution changes, because it can update its model and discard obsolete information.

However, SAND is still mainly an automatic anomaly detector. Although it is designed to adapt to streams, it does not provide an explicit user-facing mechanism for jointly exploring whether a suspicious region is a true anomaly or a drift-driven normal transition. It also does not focus on presenting the lifecycle of normal patterns to users. In contrast, the AnDri system emphasizes co-exploration: users can inspect detected anomalies, changing normal patterns, and threshold decisions in the same workflow \citep{park2025andri}.

DAMP is another important streaming method based on Matrix Profile ideas \citep{lu2022damp}. It is designed for highly scalable anomaly detection and computes exact left-discords on fast-arriving streams. Its primary strength lies in computational efficiency: it can process extremely large streams where exhaustive discord search would otherwise be computationally impractical \citep{lu2022damp}. As such, DAMP serves as a strong baseline for streaming subsequence anomaly detection.

However, in the problem setting considered by AnDri, DAMP mainly focuses on efficient discord discovery rather than explicitly modeling concept drift. Since discord-based methods rely on nearest-neighbor dissimilarity, sustained or recurring changes in normal behavior can be difficult to classify correctly using discord logic alone. Once a newly emerged pattern begins to recur frequently, its nearest-neighbor distance may decrease, causing it to no longer appear as a strong discord. Conversely, short-lived but meaningful behavioral shifts may still be flagged as anomalies even when they represent a legitimate new normal state \citep{park2025adaptive}.


\section{Concept Drift Detection and Adaptation}

Concept drift refers to changes in the underlying data distribution over time. Prior surveys describe drift adaptation as a central problem in learning from evolving data streams \citep{gama2014survey,lu2019learning}. A representative method is ADWIN, which uses an adaptive sliding window to detect distribution changes and adjust the window size according to observed evidence of drift \citep{bifet2007adwin}. Such methods are useful because they recognize that older data may become less relevant as the stream evolves.

However, concept drift detection and anomaly detection are often studied as separate tasks. Drift detection methods usually aim to identify distributional change, while anomaly detection methods aim to identify abnormal events. In real time series, these two phenomena can look similar: both appear as deviations from past behavior. If a system treats all change as anomalous, it may produce many false positives during drift. If it adapts too aggressively, it may absorb true anomalies into the normal model \citep{park2025andri,park2025adaptive}.

\section{Benchmark}

The TSB-UAD benchmark provides an end-to-end suite for evaluating univariate time-series anomaly detection methods \citep{paparrizos2022tsbuad}. It includes many datasets with labeled anomalies and supports standardized comparison across different anomaly detection algorithms. This is useful for AnDri because baseline methods such as NormA, SAND, and DAMP can be compared under a shared evaluation environment.

Nevertheless, a benchmark suite does not by itself solve the ambiguity between anomalies and drift. Benchmarks provide datasets, labels, implementations, and metrics, but the system designer still needs to decide how normal behavior should evolve over time and how users should inspect uncertain results. The AnDri system builds on TSB-UAD benchmark while adding a drift-aware and interactive workflow for anomaly exploration.

\section{Problem Domain}

A time series is a sequence of observations recorded over time. Time-series data are common in many real-world applications, such as electrocardiogram (ECG) monitoring, electricity demand forecasting, and financial markets \citep{schmidl2022anomaly}. Unlike ordinary data, the order of observations in a time series is important. The meaning of a value depends not only on its magnitude, but also on when it occurs and how it relates to surrounding observations.

The goal of time-series anomaly detection is to identify data instances or temporal segments whose behavior is inconsistent with the normal patterns in a time series \citep{chandola2009anomaly}. Anomalies can occur at different temporal scales. An anomaly may appear as a single observation, such as an isolated spike in a sensor reading. The anomaly may also span a short interval whose overall shape differs from the rest of the time series, even though no individual observation is particularly extreme. Contextual anomalies depend on the surrounding circumstances and may be considered abnormal only under specific conditions, such as an elevated heart rate during rest but not during exercise. Detecting these anomalies requires considering observed values and the context in which they occur.

Concept drift refers to a change in the underlying data distribution over time \citep{gama2014survey,lu2019learning}. In time-series anomaly detection, the definition of normal behavior may not be fixed. For example, electricity usage may change between weekdays and weekends, weather measurements may shift gradually across seasons, and ECG patterns may change when a patient's activity or physiological condition changes. A changed pattern may be either a true anomaly or an actual new normal state.

For example, in Figure~\ref{fig11}, the first region represents an old normal pattern. Later, the data gradually changes into a new normal pattern. A detector that compares all future points against only the old normal pattern may incorrectly flag the drift and new normal region as anomalous. However, a detector that adapts without control may also miss the true anomaly. This is the main challenge addressed in this report: anomalies and drift can look similar in the signal, but they require different interpretations.

Existing time-series anomaly detection methods provide important foundations for this problem. A common workflow in these methods is to first assign an anomaly score to each point, and then convert these scores into binary anomaly labels using a threshold. Choosing the threshold is difficult in practice. Since the calculation of the anomaly score of a point is dependent on the patterns in the time series. A global threshold may work for datasets which contain similar normal patterns, but it may fail when the data contain drift. When the normal behavior changes, the anomaly-score distribution may also change. As a result, a strict threshold may incorrectly classify drift-driven normal behavior as anomalous, while a loose threshold may miss true abnormal anomalies.

AnDri algorithm addresses this challenge by considering anomaly detection and concept drift detection together \citep{park2025andri,park2025adaptive}. AnDri algorithm does not use a single global threshold. It maintains multiple normal patterns and updates them as the data changed. This allows the system to distinguish anomalies from changes caused by concept drift.

In this report, \textbf{AnDri algorithm} refers to the anomaly detection algorithm \citep{park2025adaptive}. The \textbf{AnDri system} is the interactive visual analytics platform developed in this project.

\section{Project Goals}

The main objective of this project is to present the AnDri algorithm through an interactive demonstration platform. AnDri addresses the issue of anomaly detection in the context of concept drift by simulating changing normal patterns and generating anomaly scores for time series data. However, the output of the algorithm alone is not easily understandable for users. Therefore, this project has built an interactive system that enables users to intuitively explore the input time series, examine the detected anomaly candidates, observe the changes in normal behavior, and understand how AnDri distinguishes anomalies from concept drift. The second objective is to assist in "human-machine collaborative" anomaly detection through user feedback. When there is concept drift in the data, selecting the threshold for anomaly scores is difficult. Therefore, this system allows users to review the candidate results and provide feedback on whether the selected points or areas should be regarded as anomalies or normal. Then, these feedbacks will be used to adjust the detection threshold and improve the final anomaly determination results. The ultimate goal is to evaluate and showcase the workflow of the entire AnDri system. The platform integrates functions such as dataset loading, training area selection, anomaly annotation, AnDri execution, baseline comparison, candidate review, threshold adjustment, and result visualization.


\section{Contributions}

My contributions are listed below.

\begin{enumerate}

\item \textbf{Interactive visual analytics system for anomaly and drift exploration.}
Implemented a light end-to-end web-based system that supports dataset loading, training-region selection, anomaly labeling, candidate review, detector execution, and result comparison. Developed an interactive visualization interface that supports the full anomaly detection workflow, from data loading and annotation to detector execution and result comparison.

\item \textbf{Feedback-driven threshold refinement framework.}
Designed and implemented a human-in-the-loop threshold refinement workflow based on Bayesian Gaussian mixture modeling and active learning. The framework uses user feedback to refine anomaly thresholds, identify uncertain candidates, and support pattern-specific threshold assignment under concept drift.

\item \textbf{Integration and evaluation of anomaly detection methods.}
Integrated AnDri, NormA, SAND, and DAMP into a unified workflow and conducted comparative experiments using different datasets. Evaluated detection performance using precision, recall, F1-score, and AUC, and analyzed the ability of different methods to distinguish anomalies from concept drift.

\end{enumerate}
